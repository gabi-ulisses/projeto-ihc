#DICAS

Utilização da Semântica HTML

Elementos Apropriados: Utilize elementos HTML semânticos adequados como < button >, < header >, < nav >, < main >, < article >, < aside >, < footer > para descrever corretamente o propósito de cada parte do conteúdo.

Evitar < div > e < span > para funções semânticas: Evite usar

e para elementos que têm uma função específica. Por exemplo, use < button > ao invés de < div > para botões.

Estrutura de Conteúdo Textual

Títulos e Parágrafos: Estruture o conteúdo com títulos (< h1 >, < h2 >, etc.), parágrafos (< p >), e listas (< ul >, < ol >).

Hierarquia Correta: Mantenha uma hierarquia correta de títulos para facilitar a navegação e compreensão do conteúdo.

Linguagem Clara e Simples

Clareza: Utilize uma linguagem clara e evite jargões ou gírias.

Expansão de Abreviações e Acrônimos: Expanda abreviações e acrônimos ao menos na primeira vez que aparecem no texto.

Layouts de Página

Evitar Tabelas para Layout: Não use tabelas HTML para criar layouts de páginas. Utilize CSS para essa finalidade.

Semântica HTML5: Utilize a semântica HTML5 para seccionar o conteúdo (< nav >, < header >, < footer >, < main >, < article >, < section >, < aside >).

Controles de Interface do Usuário (UI)

Elementos Interativos: Use elementos nativos como < button >, < a >, < input >, < select > para interações.

Acessibilidade pelo Teclado: Certifique-se de que os elementos interativos são acessíveis pelo teclado, utilizando atributos como tabindex e eventos de teclado em JavaScript se necessário.

Rótulos de Texto Significativos

Descritivos e Específicos: Use rótulos claros e específicos para botões e links. Evite rótulos genéricos como "Clique aqui".

Formulários: Utilize o elemento < label > para associar rótulos a campos de formulário (< input >, < textarea >, etc.).

Tabelas de Dados Acessíveis

Cabeçalhos de Tabela: Utilize < th > para definir cabeçalhos de tabela e o atributo scope para especificar se são cabeçalhos de linha ou coluna.

Legenda e Resumo: Use < caption > e o atributo summary para fornecer descrições e resumos da tabela.

Alternativas em Texto

Imagens: Sempre forneça um texto alternativo (alt) para imagens. Certifique-se de que o texto seja descritivo do conteúdo da imagem.

Conteúdo Multimídia: Ofereça transcrições para conteúdo de áudio e legendas para vídeos.

Testar Acessibilidade

Leitores de Tela: Teste seus documentos utilizando leitores de tela para verificar a acessibilidade.

Ferramentas de Avaliação: Utilize ferramentas de avaliação de acessibilidade como WAVE, Axe, ou Lighthouse para identificar e corrigir problemas de acessibilidade.

Boa Prática Geral

Consistência: Mantenha consistência no uso de semântica HTML e estrutura do documento desde o início do projeto.

Educação Contínua: Mantenha-se atualizado com as melhores práticas e recomendações de acessibilidade, como as diretrizes WCAG (Web Content Accessibility Guidelines).
